## Reverse coverage — openings → puzzles (the rolodex-readiness signal)

The openings DB has **3,641** entries. Many share names across ECO codes
(e.g., "Sicilian Defense" at B20, B27, B50, B50). Deduped by name, that's
**3,149 unique-named openings**.

Match strategy: normalize each opening name (strip apostrophes / colons /
commas; spaces → underscores; diacritics removed) and check whether the
resulting token appears in any puzzle's `openingTags` (either as the
family token or as a variation token). Both views below count puzzle
hits per opening; the DB's ECO duplication is preserved in the raw

## Implications for the rolodex

### What works today (no new tagging needed)

1. **The top-25 rolodex-ready openings carry the experience.** Sicilian,
   French, Italian, Caro-Kann, Queen's Pawn, Scandinavian, English,
   QGD, Scotch, Ruy Lopez — these are the openings a typical user
   favorites and they have 70–500 puzzles each.
2. **Family-level matching is solid for the top 78–108 families.**
   Most heavily-played openings normalize cleanly into the
   Lichess family token.

### What breaks for the rolodex Puzzles row

1. **Puzzles row will be empty or thin for ~94% of DB openings.** If a
   user favorites a deep variation (e.g., "Sicilian Defense: Najdorf,
   English Attack"), the rolodex will show 0 puzzles for that card —
   even though 491 Sicilian-family puzzles exist. The rolodex needs a
   **fallback ladder**: try the exact name → try the parent family →
   show "X puzzles in this family" with the family name as the chip.
2. **Petrov's Defense shows 0 puzzles** without an alias mapping (49
   are tagged `Russian_Game`). Cheap win: hardcode a 13-entry alias
   table in the rolodex selector, recover 133 puzzles.
3. **The kid sub-400 pool (300 puzzles) is entirely opening-blind.** A
   Kid-Mode rolodex variant cannot draw opening-filtered puzzles from
   this set. Either ship Kid-Mode without per-opening puzzles or
   re-tag the training pool (separate WO).
4. **GM Games coverage is worse than puzzles, not better.** No progress
   data on `modelGames` exists at all (see WO-ROLODEX-AUDIT-01 §A2).
   The rolodex's GM Games row was already flagged for "—" treatment in
   v1; this audit confirms it.

### What the plumbing WO should do

1. **Build the rolodex Puzzles selector with a family-fallback chain**:
   - exact-name match → parent-family match → "no puzzles tagged for
     this line — try {family}" nudge copy.
2. **Add the 13-entry alias map** for the Lichess naming quirks
   (Russian_Game ↔ Petrov's Defense, etc.). Put it in
   `src/services/openingTagAliases.ts` or similar. ~30 LOC.
3. **Do not attempt to re-tag puzzles.** Lichess curates the tags;
   bulk-re-tagging would require a chess-engine pass over 11,793
   untagged puzzles and is out of scope.
4. **Defer Kid-Mode opening-filtered puzzles** until the training pool
   gets tagged. Out of scope for the rolodex v1.
